# Frontend Design Skills

## Description
Comprehensive frontend development skill covering HTML, CSS, JavaScript, and modern frontend frameworks. Focuses on creating responsive, performant, and maintainable web applications.

## Features
- Modern HTML5 and semantic markup
- Advanced CSS (Grid, Flexbox, Animations, Custom Properties)
- JavaScript (ES6+, Async programming, DOM manipulation)
- Frontend frameworks (React, Vue, Angular, Svelte)
- CSS frameworks and preprocessors (Tailwind, Bootstrap, Sass)
- Build tools and bundlers (Webpack, Vite, Rollup)
- State management and data flow
- Performance optimization
- Cross-browser compatibility
- Progressive Web Apps (PWA)

## Usage
Use this skill when you need to:
- Build responsive web applications
- Implement modern frontend architectures
- Optimize website performance
- Create interactive components and features
- Set up build pipelines and development environments
- Debug frontend issues
- Implement SEO best practices

## Tools
- Editors: VS Code, WebStorm, Sublime Text
- Browsers: Chrome DevTools, Firefox Developer Tools
- Testing: Jest, Cypress, Playwright
- Performance: Lighthouse, WebPageTest
- Version Control: Git, GitHub, GitLab

## Commands
Create a new frontend project:
```
/frontend create-project --name my-app --framework react
```

Optimize website performance:
```
/frontend optimize --target ./dist
```

Generate performance report:
```
/frontend performance-audit --url https://example.com
```

## Best Practices
- Use semantic HTML for better accessibility and SEO
- Implement responsive design with CSS Grid and Flexbox
- Optimize images and assets for faster loading times
- Use modern JavaScript features appropriately
- Follow accessibility guidelines (WCAG)
- Implement proper error handling and loading states
- Use CSS-in-JS or utility classes for maintainable styles
- Test on multiple browsers and devices
- Follow security best practices (CSP, XSS prevention)
- Implement proper caching strategies

## Performance Tips
- Minimize and compress assets
- Use lazy loading for images and components
- Implement code splitting for larger applications
- Optimize bundle size with tree shaking
- Use CDN for static assets
- Implement service workers for offline functionality
- Monitor Core Web Vitals regularly
